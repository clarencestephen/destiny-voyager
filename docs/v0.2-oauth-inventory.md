# v0.2.0 — OAuth + Inventory + Tagging

Destiny Voyager v0.2.0 adds:

- **Bungie OAuth sign-in (PKCE)** — `auth.py` opens your browser, you authorize, we get a refresh token. No client_secret needed since the Bungie app is now a Public OAuth client.
- **Full inventory fetch** — `fetch_inventory.py` pulls your vault + per-character equipped + character inventories + in-game saved loadouts (Lightfall+ native loadouts) and writes the `INVENTORY` and `MY LOADOUTS` sheets.
- **DIM-compatible tagging** — 5 tags: `favorite` / `keep` / `infuse` / `junk` / `archive`. CLI in `tag.py`.
- **Loadout sharing** — `share_loadout.py` exports loadouts to portable JSON files you can swap with friends, or import a friend's JSON back into your config.
- **DIM CSV import** — `import_dim_csv.py` reads a DIM Weapons/Armor CSV export into `INVENTORY` (for users who don't want to OAuth).

---

## End-to-end flow

```bash
# 1. Sign in (one-time — opens browser, ~30 sec)
python3 auth.py

# 2. Make sure the manifest is cached (decode_dim does this; or run it explicitly)
python3 decode_dim.py     # builds DIM LOADOUTS (FULL) + caches manifest

# 3. Fetch your full inventory
python3 fetch_inventory.py

# 4. Tag things you care about
python3 tag.py "fatebringer" --tag favorite
python3 tag.py "blue engram" --tag junk
python3 tag.py --list

# 5. Re-fetch to refresh INVENTORY with the new tags
python3 fetch_inventory.py
```

## OAuth flow details

```
You run auth.py
   ↓
Generates code_verifier (random 64-char string)
   ↓
Opens browser to https://www.bungie.net/en/OAuth/Authorize
  ?client_id=52250&response_type=code&state=<random>
  &code_challenge=<sha256(verifier)>&code_challenge_method=S256
   ↓
Bungie shows you the "authorize Destiny Voyager" page
   ↓
You click Authorize
   ↓
Bungie redirects to http://localhost:8123/callback?code=<auth_code>&state=<random>
   ↓
Destiny Voyager's local HTTP listener catches the code
   ↓
auth.py POSTs to https://www.bungie.net/Platform/App/OAuth/Token/
  grant_type=authorization_code, code=<>, code_verifier=<>, client_id=52250
   ↓
Bungie returns:
  - access_token (1 hour TTL)
  - refresh_token (90 days TTL)
  - membership_id (your Bungie account ID)
   ↓
auth.py writes oauth.{access_token, refresh_token, expires_at, membership_id}
to user_config.json (chmod 600)
```

After that, every other script calls `auth.get_valid_token()` which:
- Returns the cached access_token if it hasn't expired yet
- Otherwise POSTs the refresh_token to get a new access_token (re-using the same refresh)
- If the refresh_token itself has expired (90 days), you'll get an error and need to re-run `python3 auth.py`

## Inventory data shape

`fetch_inventory.py` writes the INVENTORY sheet with these columns:

| # | Name | Tier | Element | Type | Slot | Power | Tag | Location | Instance ID |
|---|---|---|---|---|---|---|---|---|---|
| 1 | Crimson | Exotic | Solar | Hand Cannon | Kinetic | 1810 | favorite | Hunter EQUIPPED | 6917530...|
| 2 | Conditional Finality | Exotic | Stasis | Shotgun | Energy | 1820 | keep | Vault | 6917530...|
| 3 | Smite of Merain | Legendary | Arc | Pulse Rifle | Energy | 1750 | junk | Vault | 6917530...|

You can sort/filter in Excel by:
- **Tag** to see all your favorites
- **Location** to see what's vaulted vs equipped vs on a specific character
- **Power** descending to find your highest-power items
- **Tier** + **Type** to compare all your hand cannons

## Tag system

5 tags, DIM-compatible (so if you've already tagged in DIM, exporting their CSV via `import_dim_csv.py` brings tags over):

| Tag | Color | Meaning |
|---|---|---|
| `favorite` | gold | Top-tier rolls, never delete |
| `keep` | green | Solid roll, keep around |
| `infuse` | yellow | Infusion fuel — power-up material |
| `junk` | red | Flagged to dismantle |
| `archive` | grey | Out of meta but want to remember |

Tags live in `user_config.json` under `item_tags`, keyed by instance ID:

```json
"item_tags": {
  "6917530102879201062": "favorite",
  "6917530019751607231": "keep",
  ...
}
```

They persist across `fetch_inventory.py` runs — the sheet is rewritten but tags survive in the config.

## Loadout sharing

Export an in-game loadout to JSON:

```bash
python3 share_loadout.py --export-game --char Hunter --slot 3 \
    --name "Crota DPS" --out loadouts/crota_dps.json
```

Friends import:

```bash
python3 share_loadout.py --import loadouts/crota_dps.json
```

The imported loadout lands in `user_config.json` under `saved_loadouts[]`. Re-run `fetch_inventory.py` to see it in `MY LOADOUTS` sheet.

For DIM share URLs:

```bash
python3 share_loadout.py --export-dim https://dim.gg/xxx/Raid \
    --name "My friend's build" --out loadouts/imported.json
```

## DIM CSV import path

For users who don't want to OAuth:

```bash
# Export from DIM: Settings → Spreadsheets → Weapons / Armor
python3 import_dim_csv.py ~/Downloads/destinyWeapons.csv ~/Downloads/destinyArmor.csv
```

This reads the CSV and writes the same INVENTORY sheet — with the bonus that DIM's existing tags are imported into `user_config.json` and merged with anything you've tagged via `tag.py`.

## What's NOT in v0.2.0

- **Equipping items from the toolkit** (would require `MoveEquipDestinyItems` scope; v0.3 work)
- **A real GUI** for browsing inventory/tagging (v0.3 — local Flask web app)
- **Auto-sync to DIM Sync** (DIM doesn't expose a public sync API)
- **Token storage in OS keyring** (currently chmod-600 file; can add `keyring` package later)

## Security notes

- The Bungie app is now a Public OAuth client → no `client_secret` needed in the .exe
- `access_token` has 1h TTL — short-lived
- `refresh_token` has 90d TTL — long-lived but revocable from your Bungie account settings
- Both live in `user_config.json` which is `.gitignore`d and chmod-600
- The `localhost:8123` callback is local-only — no external traffic
